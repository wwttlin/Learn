# 補習班管理系統 - 快速開始

## 🎯 系統功能

這是一個完整的補習班管理系統，包含以下核心功能：

### 📊 系統總覽
- 學生數量統計
- 課程數量統計  
- 月收入統計
- 待繳費提醒

### 👥 學生管理
- **完整學生資料**：姓名、英文名字、出生年月日、就讀學校班級
- **聯絡資訊**：學生電話、電子郵件、地址
- **家長資訊**：家長姓名、家長電話
- **學籍管理**：入學日期、學生狀態（在學/停學）
- **搜尋功能**：可搜尋姓名、英文名字、學校班級、電話、家長姓名
- **完整操作**：新增、編輯、刪除學生資料

### 📚 課程管理
- 建立課程資料
- 設定不同繳費方式的價格：
  - 月繳
  - 季繳  
  - 半年繳
- 編輯和刪除課程
- 課程描述管理

### 💰 繳費管理
- **費用項目管理**：每筆繳費可設定不同的費用項目（學費、教材費、註冊費等）
- **日期管理**：記錄每筆費用的對應日期
- **多項目支援**：一個學生可以有多個不同的費用項目
- **訂金制度**：支援預收訂金，後續繳納尾款
- **分階段繳費**：清楚顯示訂金階段、尾款階段、已完成等狀態
- **多種付款方式**：現金、轉帳、信用卡、支票
- **繳費追蹤**：即時顯示總金額、已繳金額、剩餘金額
- **繳費明細**：記錄每次繳費的詳細資訊
- **靈活操作**：可刪除錯誤記錄，支援備註功能

## 🚀 如何啟動

### 1. 安裝依賴
```bash
npm install
cd client && npm install && cd ..
```

### 2. 啟動系統
```bash
# 方法一：同時啟動前後端
npm run dev

# 方法二：分別啟動
npm run server  # 後端 API (Port 5000)
npm run client  # 前端界面 (Port 3000)
```

### 3. 訪問系統
- 前端界面：http://localhost:3000
- 後端 API：http://localhost:5000

## 📝 使用流程

1. **建立課程**：先在「課程管理」中新增課程和價格
2. **新增學生**：在「學生管理」中登記學生資料
3. **記錄繳費**：在「繳費管理」中記錄學生繳費情況
4. **查看總覽**：在「系統總覽」中查看整體營運狀況

## 🔧 主要功能操作

### 學生管理
- **搜尋**：在搜尋框輸入學生姓名、英文名字、學校班級、電話或家長姓名
- **新增學生**：
  - 基本資料：中文姓名（必填）、英文名字、出生年月日
  - 學校資訊：就讀學校班級（例如：台北市立中山國小五年三班）
  - 聯絡資訊：電話、電子郵件、地址
  - 家長資訊：家長姓名、家長電話
- **編輯**：點擊學生列表中的「編輯」按鈕修改所有資料
- **刪除**：點擊「刪除」按鈕（有繳費記錄的學生無法刪除）
- **狀態管理**：可設定學生為「在學」或「停學」

### 課程管理
- **新增課程**：設定課程名稱、描述和各種繳費方式的價格
- **編輯課程**：修改課程資訊和價格
- **刪除課程**：刪除不需要的課程（有繳費記錄的課程無法刪除）

### 繳費管理
- **新增繳費記錄**：
  - 選擇學生和課程
  - 輸入費用項目（例如：2025年1月學費、教材費、註冊費）
  - 設定費用日期
  - 系統自動計算總金額
  - 可設定訂金金額（可為0，表示全額繳費）
  - 系統自動計算剩餘金額
- **繳納尾款**：
  - 點擊「繳尾款」按鈕
  - 可選擇付款方式（現金、轉帳、信用卡、支票）
  - 支援部分繳費和備註功能
- **繳費狀態**：
  - **待繳費**：尚未繳納任何費用
  - **部分繳費**：已繳納訂金，尚有尾款未繳
  - **已完成**：已繳納全部費用
- **繳費階段**：
  - **訂金階段**：剛建立記錄，等待繳納訂金
  - **尾款階段**：已繳訂金，等待繳納尾款
  - **全額繳費**：一次性繳納全部費用
  - **已完成**：所有費用已繳清

## 💡 系統特色

- **簡單易用**：直觀的操作界面
- **自動計算**：根據繳費方式自動計算金額
- **即時統計**：實時顯示營運數據
- **資料完整**：完整記錄學生和繳費資訊

## 🔧 技術架構

- 前端：React + TypeScript
- 後端：Node.js + Express
- 資料庫：SQLite（自動建立）
- 樣式：自定義 CSS

## 📁 資料儲存

系統會自動在專案根目錄建立 `tutoring.db` 檔案來儲存所有資料。

## 💡 繳費管理使用範例

### 情境：學生張小明報名數學課程，需要繳納多項費用

#### 範例1：2025年1月學費
1. **建立繳費記錄**：
   - 選擇學生：張小明
   - 選擇課程：數學課程
   - 費用項目：2025年1月學費
   - 費用日期：2025-01-01
   - 繳費方式：季繳
   - 總金額：NT$ 20,000（自動計算）
   - 訂金金額：NT$ 3,000
   - 系統自動計算剩餘金額：NT$ 17,000

#### 範例2：教材費
1. **建立另一筆繳費記錄**：
   - 選擇學生：張小明
   - 選擇課程：數學課程
   - 費用項目：數學教材費
   - 費用日期：2025-01-15
   - 繳費方式：月繳
   - 總金額：NT$ 1,500（自動計算）
   - 訂金金額：NT$ 1,500（全額繳費）

2. **繳納尾款**：
   - 當學生正式開始上課後
   - 點擊該記錄的「繳尾款」按鈕
   - 輸入繳費金額：NT$ 17,000
   - 選擇付款方式：轉帳
   - 添加備註：正式開課後尾款
   - 確認繳費

3. **狀態變化**：
   - 初始：訂金階段 → 部分繳費
   - 繳納尾款後：已完成 → 已完成

## ⚠️ 注意事項

- 請定期備份 `tutoring.db` 檔案
- 首次使用請先建立課程再新增學生
- 訂金金額不能超過總金額
- 有繳費記錄的學生和課程無法刪除
- 系統會自動建立所需的資料表
